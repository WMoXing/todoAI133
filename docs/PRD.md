# AI 增强 Todo — 产品需求文档 v1.0

> 2026-06-01 | 单人项目 | V1 + V2 分期交付

---

## 问题陈述

现有 todo 工具的痛点：录入繁琐（手动填标题、选日期、加标签）、大任务不知如何拆解导致拖延、缺乏 AI 驱动的任务管理视角。我需要一个 **AI 深度嵌入** 的个人 todo 工具——自然语言输入即可管理任务，AI 帮我拆解复杂任务、跟我聊进度、做复盘。

---

## 解决方案

一个 Web 应用，双栏布局：左侧任务管理区（列表+看板切换），右侧 AI 对话面板。DeepSeek API 驱动 AI 能力，分两期交付。数据用 JSON 文件存缤纷云对象存储，后端 Express.js 部署在 Render，前端 Vue 3 + Element Plus。

- **V1（核心 AI 体验）**：自然语言录入 → 自动解析为结构化任务；AI 拆解复杂任务 → 预览确认 → 生成子任务；侧边栏对话交互，持久化记忆。
- **V2（智能增强）**：AI 智能排程（今天该做什么）、每日/每周 AI 复盘报告。

## 用户故事

### V1 — 核心功能

1. 作为用户，我希望输入一句自然语言（如"明天下午3点去超市买菜，很重要"），系统自动解析出标题、截止时间、优先级和标签，这样我就能快速录入任务，不用逐个填表单。
2. 作为用户，我希望手动创建任务，字段包括标题、截止日期/时间、标签、优先级（高/中/低）和备注，这样我可以完全掌控任务细节。
3. 作为用户，我希望勾选任务标记为完成，这样我可以跟踪进度。
4. 作为用户，我希望编辑和删除任务，这样我的列表始终是最新的。
5. 作为用户，我希望删除的任务进入回收站（保留30天），这样我可以恢复误删的任务。
6. 作为用户，我希望将任务组织到不同的清单/项目中（如"工作"、"学习"、"购物"），这样我可以区分不同的生活领域。
7. 作为用户，我希望设置重复任务（如"每周五写周报"），这样周期性任务不用手动重新创建。
8. 作为用户，我希望按标题搜索任务，按标签、优先级或状态筛选，这样我能快速找到特定任务。
9. 作为用户，我希望在列表视图（默认）和看板视图（待办/进行中/已完成）之间切换，这样我可以选择适合当前工作流的方式查看任务。
10. 作为用户，我希望在看板视图拖拽任务来改变状态，这样我可以直观地管理任务流转。
11. 作为用户，我希望切换深色模式，这样晚上使用应用时眼睛更舒适。
12. 作为用户，我希望应用在手机上响应式适配（单栏，底部 Tab 切换"任务"和"AI 助手"），这样我可以在手机上管理任务。
13. 作为用户，我希望选中一个任务后让 AI 帮我拆解成子步骤，预览 AI 生成的拆解结果，可以增删改子步骤，确认后生成子任务，这样我面对复杂任务时不用自己想办法分步。
14. 作为用户，我希望右侧有一个持久化的 AI 对话面板，可以跟 AI 聊我的任务（如"这周还有哪些没做完"、"总结一下我的进度"），这样我随时都有个任务助手。
15. 作为用户，我希望 AI 对话历史跨会话持久化，这样 AI 能记住上下文，我可以继续之前的对话。
16. 作为用户，我希望 AI 回复以打字机效果逐字流式输出，这样等待长推理时有视觉反馈，不会焦虑。
17. 作为用户，我希望给任务添加 Markdown 格式的备注，这样我可以记录任务的详细信息。
18. 作为用户，我希望支持父子任务关系（子任务），这样我可以跟踪复杂任务的部分进度。

### V2 — 智能增强

19. 作为用户，我希望 AI 根据截止时间、优先级和我的使用习惯，每天推荐"今日聚焦三件事"的优先级列表，这样我知道每天该先做什么。
20. 作为用户，我希望 AI 生成每日/每周复盘报告，包含完成率、时间分析和改进建议，这样我可以反思并提高效率。

## 实现决策

### 架构

- **前端**：Vue 3 Composition API（`<script setup>`）+ Vite + Element Plus（UI 组件库）
- **后端**：Express.js（Node.js），部署在 Render
- **数据库**：单个 JSON 文件，存储在缤纷云对象存储（S3 兼容）；服务启动时加载到内存，写入时持久化回缤纷云
- **AI 引擎**：DeepSeek API（流式输出）
- **文件附件**：缤纷云对象存储（同桶，不同 key 前缀）

### 数据模型

**Task（任务）**：
- `id`：唯一标识
- `title`：标题
- `completed`：是否完成
- `status`：状态 —— `"todo"` | `"in_progress"` | `"done"`
- `dueDate`：截止日期，ISO 格式（可为空）
- `dueTime`：截止时间（可为空）
- `tags`：标签数组
- `priority`：优先级 —— `"high"` | `"medium"` | `"low"`
- `notes`：Markdown 备注
- `parentId`：父任务 ID（可为空，用于子任务关系）
- `childIds`：子任务 ID 数组
- `projectId`：所属清单 ID
- `createdAt`：创建时间
- `updatedAt`：更新时间
- `deletedAt`：软删除时间（可为空，非空表示在回收站，30天自动清除）
- `isRecurring`：是否重复任务
- `recurrenceRule`：重复规则 —— `{ frequency: "daily"|"weekly"|"monthly", interval: number, daysOfWeek?: number[] }`（可为空）
- `aiGenerated`：是否 AI 生成
- `aiParentId`：AI 拆解的来源父任务 ID（可为空）

**Project（清单/项目）**：
- `id`：唯一标识
- `name`：名称
- `color`：颜色（hex）
- `createdAt`：创建时间

**ChatSession（对话会话）**：
- `id`：唯一标识
- `messages`：消息数组 —— `{ role: "user"|"assistant", content: string, timestamp: string }`
- `contextTaskIds`：对话中引用的任务 ID 数组
- `createdAt`：创建时间
- `updatedAt`：更新时间

### AI 交互流程

**自然语言录入**：
1. 用户在任务输入框中输入自然语言
2. 前端将文本发送到后端 → 后端调用 DeepSeek API，附上解析提示词
3. AI 返回结构化 JSON：`{ title, dueDate?, dueTime?, priority?, tags? }`
4. 前端用解析结果预填创建任务表单
5. 用户可编辑修改后再确认创建

**任务拆解**：
1. 用户选中一个任务 → 点击"AI 拆解"
2. AI 生成子步骤建议，展示在预览卡片中
3. 用户可增删改子步骤
4. 确认后，所有子步骤作为子任务挂在父任务下
5. 父任务显示进度指示（如"3/6 子任务已完成"）

**AI 对话面板**：
1. 用户在右侧聊天面板发送消息
2. 后端在系统提示词中包含相关任务上下文（当前选中任务、最近任务等）
3. AI 通过 SSE（Server-Sent Events）逐 token 流式返回
4. 前端利用 Vue 响应式渲染打字机效果

### API 设计

所有接口前缀 `/api`：

- `GET /api/tasks` —— 获取任务列表（查询参数：projectId、status、tag、priority、search）
- `POST /api/tasks` —— 创建任务
- `PUT /api/tasks/:id` —— 更新任务
- `DELETE /api/tasks/:id` —— 软删除任务
- `POST /api/tasks/:id/restore` —— 从回收站恢复
- `GET /api/projects` —— 获取清单列表
- `POST /api/projects` —— 创建清单
- `PUT /api/projects/:id` —— 更新清单
- `DELETE /api/projects/:id` —— 删除清单
- `POST /api/ai/parse` —— 自然语言解析为任务结构
- `POST /api/ai/breakdown` —— 生成任务拆解
- `POST /api/ai/chat` —— AI 对话（SSE 流式）
- `GET /api/ai/chat/sessions` —— 获取对话会话列表
- `GET /api/ai/chat/sessions/:id` —— 获取指定会话的消息记录

### UX 决策

- **双栏布局**：任务区（左侧，约65%）+ AI 对话面板（右侧，约35%），对话面板可收起
- **列表视图**：默认视图，按清单分组显示任务，支持内联编辑
- **看板视图**：三列（待办 / 进行中 / 已完成），支持拖拽切换状态
- **移动端适配**：单栏布局，底部 Tab 栏切换"任务"和"AI"两个页面
- **深色模式**：顶部开关切换，偏好存 localStorage，使用 Element Plus 深色主题
- **回收站**：侧边栏入口，显示已删除任务，支持恢复和永久删除，30 天自动清理
- **重复任务**：完成任务后，根据重复规则自动生成下一次实例，更新截止日期

## 测试策略

作为个人工具，MVP 阶段不需要正式的测试套件。但以下关键路径需要手动验证：

- JSON 文件在缤纷云上的读写完整性（数据不丢失）
- DeepSeek API 流式响应及错误处理
- 中文自然语言解析的准确率
- 任务拆解生成质量
- 移动端响应式布局在各断点下的表现
- 深色模式在所有组件上的切换效果
- 软删除 → 回收站 → 30天自动清除的完整流程

## 不在范围内

- **用户认证/多用户**：单人使用，无需登录系统
- **协作/分享**：无团队功能，无任务分享
- **第三方集成**：无日历同步、Slack/微信机器人、邮件提醒
- **浏览器通知推送**：Render 免费层不支持后台定时任务
- **快捷键**：V1 不做
- **文件附件**：V2 再做，缤纷云基础设施已就绪，仅 UI 层面延后
- **离线支持 / PWA**：暂不考虑
- **从其他工具导入数据**：暂不规划

## 补充说明

- **缤纷云用法**：后端通过预配置的 API Key 读写 JSON 数据库文件。JSON 文件是唯一数据源——服务启动时加载，每次数据变更后持久化。
- **Render 兼容性**：Express 服务必须无状态，仅依赖缤纷云上的 JSON 文件。不使用无法从 JSON 重建的内存缓存。
- **DeepSeek 成本**：注意监控 API 用量。单人使用场景下，费用预计较低（约 10-30 元/月）。
- **未来扩展**：JSON 文件方案支撑几千条任务绰绰有余。如果任务量超过约 10,000 条，迁移到 SQLite 或云数据库非常简单，因为数据模型简洁。
- **语言**：UI 和 AI 提示词使用简体中文。代码注释和变量名使用英文。
